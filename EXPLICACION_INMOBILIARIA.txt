## 2. BACKEND (server.js)

### 2.1. Dependencias y configuración
- **express**: Framework para crear el servidor HTTP y las rutas.
- **mysql2/promise**: Cliente MySQL con soporte para promesas.
- **cors**: Permite peticiones desde el frontend (CORS).
- **dotenv**: Carga variables de entorno desde .env.

### 2.2. Inicialización
- Se crea una app Express.
- El puerto se toma de la variable de entorno `PORT` o por defecto 3001.
- Se configura CORS para aceptar peticiones del frontend (por defecto http://localhost:5173).
- Se habilita el parseo de JSON en los requests.

### 2.3. Conexión a la base de datos
- Se crea un pool de conexiones MySQL usando variables de entorno o valores por defecto.
- Se verifica la conexión al iniciar el servidor.

### 2.4. Helpers (funciones auxiliares)
- **mapOwner(row)**: Transforma una fila de la tabla personas+propiedades al formato esperado por el frontend para propietarios.
- **mapTenant(row)**: Igual que arriba pero para inquilinos.
- **mapProperty(row)**: Igual para propiedades.
- **mapLease(row)**: Igual para contratos.
- **mapTipoToDB(tipo)**: Traduce el tipo de propiedad del frontend al valor ENUM de la base de datos.
- **mapTipo(tipo)**: Traduce el tipo de la base de datos al formato amigable para el frontend.

### 2.5. Endpoints principales

#### PROPIEDADES
- **GET /api/properties**: Lista todas las propiedades activas, con leaseId si tiene contrato activo.
- **POST /api/properties**: Crea una nueva propiedad. Separa dirección y número, asigna estado según si está ocupada o no.
- **PUT /api/properties/:id**: Actualiza una propiedad existente.
- **DELETE /api/properties/:id**: Baja lógica (no borra, solo marca como inactiva).

#### PROPIETARIOS
- **GET /api/owners**: Lista propietarios activos y sus propiedades.
- **POST /api/owners**: Crea un propietario. Separa nombre y apellido.
- **DELETE /api/owners/:id**: Baja lógica.

#### INQUILINOS
- **GET /api/tenants**: Lista inquilinos activos y leaseId si tiene contrato activo.
- **POST /api/tenants**: Crea un inquilino.
- **DELETE /api/tenants/:id**: Baja lógica.

#### CONTRATOS
- **GET /api/leases**: Lista contratos activos o vencidos (no borradores).
- **POST /api/leases**: Crea un contrato, marca la propiedad como alquilada.
- **DELETE /api/leases/:id**: Elimina contrato y libera la propiedad si no tiene otro contrato activo.
- **PATCH /api/leases/:id/status**: Cambia el estado del contrato (activo, vencido, rescindido, renovado).

#### OTROS
- **GET /api/health**: Healthcheck simple.

### 2.6. Variables y funciones importantes
- **pool**: Pool de conexiones MySQL, usado en todos los queries.
- **app**: Instancia de Express.
- **PORT**: Puerto de escucha.
- **mapOwner, mapTenant, mapProperty, mapLease**: Transforman datos de la BD al formato frontend.
